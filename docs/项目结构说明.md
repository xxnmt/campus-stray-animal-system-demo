# 校园流浪动物系统 - 项目结构说明文档

## 项目概述

**项目名称**：笃行猫谱（campus-stray-animal-system）  
**当前版本**：v0.0.6  
**技术栈**：微信小程序 + EMAS（阿里云 Serverless）  
**功能**：记录校园流浪猫的信息、照片、评分，为流浪猫建立"档案"

---

## 目录结构说明

### 1. LafToEMAS/ - 数据迁移工具
```
LafToEMAS/
├── json/           # 原始 JSON 数据文件（从 Laf 云开发导出）
│   ├── badge.json
│   ├── cat.json
│   ├── comment.json
│   ├── feedback.json
│   ├── photo.json
│   ├── user.json
│   └── ... (其他数据表)
└── processJson.js  # 数据处理脚本
```
**功能**：将项目从 Laf 云开发平台迁移到 EMAS 阿里云 Serverless 平台的数据转换工具。

---

### 2. functionsEMAS/ - 云函数代码
```
functionsEMAS/
├── temp/               # 临时云函数
│   ├── tempRenameCatNo/
│   ├── transferFromLaf/
│   └── updateEMAS/

│   ├── index.js
│   ├── countPhoto.js       # 统计照片

│   └── getTempCOS.js       # 获取临时 COS 凭证
└── unionOp/           # 统一操作云函数（主入口）
    ├── index.js       # 统一调度入口
    ├── _helper.js     # 辅助函数
    ├── userOp.js      # 用户操作（登录、信息更新）
    ├── curdOp.js      # 数据库 CRUD 通用操作
    ├── catRelationOp.js    # 猫咪关系管理
    ├── commentCheck.js     # 评论审核
    ├── managePhoto.js      # 照片管理
    ├── updateCat.js        # 猫咪信息更新
    ├── getBadge.js         # 获取徽章
    ├── giveBadge.js        # 发放徽章
    ├── vaccineOp.js        # 疫苗管理
    ├── getURL.js           # 获取上传 URL
    ├── getTempCOS.js       # 获取 COS 临时凭证

```
**功能**：所有后端业务逻辑，包括用户管理、猫咪信息管理、照片审核、徽章系统、疫苗记录等。

## 核心功能模块

### 猫咪管理
- 猫咪信息展示（详情页）
- 添加猫咪照片
- 猫咪评分
- 猫咪关系图谱

### 用户系统
- 用户注册/登录
- 个人信息管理（头像、昵称）
- 关注猫咪
- 用户徽章系统

### 内容管理
- 新闻发布
- 科普文章
- 评论系统
- 照片审核

### 管理后台
- 照片审核
- 猫咪信息管理
- 反馈处理
- 徽章管理
- 管理员权限

### AI 识猫
- 拍照识猫（分包）
- 根据图片识别相似猫咪

---

## 技术架构

**前端**：微信小程序原生框架 + WeUI 组件库  
**后端**：EMAS Serverless（云函数 + 云数据库 + 云存储）  
**数据库**：EMAS 云数据库（MongoDB 兼容）  
**存储**：EMAS 内置云存储（原使用腾讯云 COS，现已禁用）
